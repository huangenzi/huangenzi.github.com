移动端1像素问题

发表于 2017-03-06

设备像素(device pixel, dp)
设备像素是物理概念，又称物理像素，指设备能控制显示的最小物理单位，即显示器上一个个的点。从屏幕在工厂生产出的那天起，它上面设备像素点就固定不变了。比如iphone5的分辨率为640x1136px

css像素(css pixel, px)
css像素是Web编程的概念，指的是CSS样式代码中使用的逻辑像素。在CSS规范中，长度单位可以分为两类，绝对(absolute)单位以及相对(relative)单位。px是一个相对单位，相对的是前面所说的设备像素(device pixel)。比如iPhone5的CSS像素为320x568px.

导致1像素问题的原因
UI设计师要求的1px是指物理像素1px，而4开发人员CSS里写的1px是逻辑像素。它们之间不一定相等，而是有比例关系。
在javascript中可以用window.devicePixelRatio来获取，也可以用媒体查询的-webkit-min-device-pixel-ratio来获取。一些手机上devicePixelRatio等于2或者3,所以1px的线实际可能会放大为2px或者3px，所以在手机上显得较粗。

解决办法
1.用css渐变模拟(最好都加浏览器前缀，以免兼容性问题)

优点： 效果和兼容性比较好，颜色可配
缺点： 代码多，无法实现圆角

2.border-image实现
先切模拟边框的图，然后css中引入

优点：兼容性好，可实现圆角
缺点：不同颜色边框需要不同图片，圆角可能模糊

3.用box-shadow模拟

优点： 代码简洁
缺点： 颜色不好掌握，效果差强人意

4.viewport&&rem方案
利用viewport + rem + js动态的修改页面的缩放比例，实现小于1像素的显示
先在html头部引入meta

然后在js中加入动态修改缩放比和实现rem根元素字体大小的设置。

优点：github上有相对稳定的库，实现效果很好
缺点：需调整的地方比较多，相对比较适合新项目

  1. IOS8以上可以直接使用0.5px

优点：简单方便，原生支持圆角
缺点：兼容性不好

6.用缩放实现
主要是用:after，:before伪元素和transform:scale(0.5)来实现

优点: 灵活，可实现圆角
缺点: 可能会与已有样式冲突

总结： 如果是新项目，推荐viewport&&rem方案，如果是老项目做改造，推荐缩放实现。
