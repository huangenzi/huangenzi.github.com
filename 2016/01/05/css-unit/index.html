css几种常见长度单位

发表于 2016-01-05

px

像素（计算机屏幕上的一个点），一般使用”px”来设置文本，因为他比较稳定和精确。但是他是固定的，所以在一些响应式布局中不太适用

em

1em等于当前的字体尺寸
“em”是一个相对于父元素的值

rem

W3C官方描述是”font size of the root element”，即rem是相对于跟元素的

字体大小设置方案

一般我们的字体大小设置方案是，body{font-size: 62.5%;},为什么是62.5%呢？这主要是为了方便em与px的相互转换,浏览器默认的字体大小为16px，所以em的初始值为1em=16px, 这样的话12px = 0.75em, 14px = 0.875em,每次换算都很麻烦。 因为16px=100%,所以10px=62.5%,body设置了font-size: 62.5%，那么1em = 10px, 这样的话12px=1.2em，14px=1.4em，这样换算就一目了然了。

下面解释一个疑问，就是，为什么font-size用62.5%而不直接用10px?

首先，如果设置为10px，那么在IE6下就不能缩放了
那对于现代浏览器呢？
主要是兼容性和未来发展趋势的综合考虑，px这个单位的含义已经越来越混乱，几乎无法评估以后的设备时会一直像现在这样对网页的px做兼容处理，还是让px回归“像素”的本意，但62.5%代表默认字体尺寸的62.5%这个含义基本是不会乱的。

响应式Web设计离不开百分比，但是有时候百分比也不是最佳的解决方案，因为CSS的宽度是相对于包含它的最近的父元素的宽度的。有的时候呢，你就是想用视口(viewpoint)的宽度或者高度，而不是父元素的，这个时候呢我们就可以用vh和vw

vw

相对于视口的宽度,1vw等于1/100的视口的宽度，例如，浏览器的宽度为1024px,那么1vw = 10.2px
兼容性

vh

相对于视口的高度，1vh等于1/100的视口高度，例如，浏览器的高度为768px,那么1vh = 7.68px
兼容性

vmax

相对于视口高度和宽度的最大值，例如，浏览器的宽度为1000px,高度为800px，那么vmax = 10px
兼容性

vmin

相对于视口高度和宽度的最小值，例如，浏览器的宽度为1000px,高度为800px，那么vmax = 8px
兼容性

ex

小写字母”x”的高度

ch

字符0的宽度

ex和ch一般用于板式微调，比如上角标和下角标
