常用git命令整理

发表于 2016-04-09

系统配置
git config –list 显示当前的Git配置
git config -e [–global] 编辑Git配置文件
git config [–global] user.name 查看当前用户名

git config [–global] user.name “[name]” 设置用户名
git config [–global] user.email 查看当前邮箱
git config [–global] user.email “[email address]” 设置邮箱

文件操作
git init 初始化git仓库（ 把当前目录变成Git可以管理的仓库）
git add [file] 把文件添加到仓库
git rm [file] 删除工作区文件
git mv [file-original] [file-renamed] 文件改名
git commit [file] -m [message] 把文件提交到仓库，并添加备注

查看信息
git status 显示工作区状态
git diff [file] 查看修改内容
git diff HEAD – [file] 查看工作区和版本库里面最新版本的区别
git diff –cached [file] 显示暂存区和上一个commit的差异
git log 查看提交历史
git log –graph 查看分支合并图
git log –graph –pretty=oneline –abbrev-commit 查看分支合并情况（格式更美观）
git reflog 查看命令历史

回退版本和撤销操作
git reset [file] 重置暂存区的指定文件
git checkout – [file] 丢弃工作区的修改
git reset –hard HEAD 返回到上一个版本
git reset –hard commit_id 回退到指定的版本

远程仓库操作
git remote add origin git@server-name:path/repo-name.git 关联一个远程仓库
git remote -v 显示所有远程仓库
git remote show ［remote］ 显示某个远程仓库的信息
git remote add [shortname] [url] 增加一个新的远程仓库并命名
git checkout -b branch-name origin/branch-name 在本地创建和远程分支对应的分支
git branch –set-upstream branch-name origin/branch-name 建立本地分支和远程分支的关联
git clone ［url］ 克隆
git pull 抓取远程的最新提交
git push [remote] [branch] 上传本地指定分支到远程仓库
git push [remote] –force 强行推送当前分支到远程仓库，即使有冲突
git push [remote] –all 推送所有分追到远程仓库

分支操作
git branch 查看分支
git branch [name] 创建分支
git checkout [name] 切换分支
git checkout -b [name] 创建＋切换分支
git merge [name] 合并某分支到当前分支
git branch -d [name] 删除分支

标签操作
git tag 查看所有标签
git tag [name] 创建新标签
git tag -a [tagname] -m [“message”] 指定标签信息
git tag -s [tagname] -m [“message”] 可以用PGP签名标签
git push origin [tagname] 推送一个本地标签
git push origin –tags 推送全部未推送过的本地标签
git tag -d [tagname] 删除一个本地标签
git push origin :refs/tags/[tagname] 删除一个远程标签
